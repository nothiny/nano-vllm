                    ┌──────────────────────────────────────────────────────────┐

   add_request()    │   ┌─────────┐    ┌────────────┐    ┌──────────┐          │
  ─────────────────►│   │ WAITING │───►│ PREFILLING │───►│ DECODING │──► Done  │
                    │   └─────────┘    └────────────┘    └──────────┘          │
                    │        │              │                  │               │

## Prefill-First 调度流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                       schedule() - Prefill First 策略                        │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
                    ┌─────────────────────────────────────┐
                    │   has_prefill_work()?               │
                    │   (waiting 或 prefilling 非空?)      │
                    └─────────────────────────────────────┘
                           │Yes                    │No
                           ▼                       ▼
              ┌────────────────────────┐  ┌────────────────────────┐
              │  _schedule_prefill()   │  │  has_decode_work()?    │
              │                        │  │  (decoding 非空?)       │
              │  返回 prefill batch    │  └────────────────────────┘
              └────────────────────────┘         │Yes        │No
                                                 ▼           ▼
                                    ┌──────────────────┐  ┌──────┐
                                    │ _schedule_decode │  │ IDLE │
                                    │                  │  └──────┘
                                    │ 返回 decode batch│
                                    └──────────────────┘
```

## Prefill-First 时间线示例

```
假设: 4个请求 (Seq1, Seq2, Seq3, Seq4), 每个需要生成3个token

时间 ────────────────────────────────────────────────────────────────────────►

│ Prefill Phase                    │ Decode Phase                           │
│◄────────────────────────────────►│◄───────────────────────────────────────►
│                                  │                                         │
│ ┌─────────────────────────────┐  │ ┌──────┐ ┌──────┐ ┌──────┐             │
│ │ Prefill Batch 1             │  │ │Decode│ │Decode│ │Decode│             │
│ │ [Seq1, Seq2, Seq3, Seq4]    │  │ │Batch1│ │Batch2│ │Batch3│             │
│ │                             │  │ │ ALL  │ │ ALL  │ │ ALL  │             │
│ │ 填充所有序列的 KV Cache      │  │ │ 4seq │ │ 4seq │ │ 4seq │  ──► Done   │
│ └─────────────────────────────┘  │ └──────┘ └──────┘ └──────┘             │
│                                  │   tok1    tok2     tok3                │

优点:
✓ Prefill 大批量处理，GPU利用率高
✓ Decode 阶段所有序列一起解码，batch size 最大化
✓ 总吞吐量最高
```

## 与 Decode-First 策略对比

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         Prefill-First (推荐用于吞吐量)                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [P1 P2 P3 P4] ──► [D D D D] ──► [D D D D] ──► [D D D D] ──► Done           │
│   大batch          大batch       大batch       大batch                      │
│   prefill          decode        decode        decode                       │
│                                                                             │
│  特点: 少量大batch，GPU利用率高，吞吐量最大化                                │
│  适用: 批量处理、离线推理                                                    │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│                         Decode-First (推荐用于低延迟)                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [P1] ─► [D1] ─► [P2] ─► [D1,D2] ─► [P3] ─► [D1,D2,D3] ─► ...              │
│  小batch 小batch 小batch  稍大      小batch    更大                          │
│                                                                             │
│  特点: 快速返回首token (TTFT低)，但总吞吐量较低                              │
│  适用: 交互式应用、聊天机器人                                                │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 核心数据结构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              Sequence                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│  seq_id: int                    # 序列唯一标识                               │
│  status: SequenceStatus         # WAITING/PREFILLING/DECODING/FINISHED      │
│  token_ids: list[int]           # 所有token (prompt + generated)            │
│  num_prompt_tokens: int         # prompt长度                                 │
│  num_prefilled_tokens: int      # 已prefill的token数 (chunked prefill用)    │
│  block_table: list[int]         # KV Cache block映射                        │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│                             BlockManager                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│  blocks: list[Block]            # 所有KV Cache blocks                        │
│  free_block_ids: deque[int]     # 空闲block队列                              │
│  hash_to_block_id: dict         # prefix caching 哈希表                      │
│                                                                             │
│  allocate(seq) ──► 为序列分配blocks                                          │
│  deallocate(seq) ──► 释放序列的blocks                                        │
│  can_append(seq) ──► 检查是否可以追加新token                                 │
└─────────────────────────────────────────────────────────────────────────────┘
```
